Appenders

The following core appenders are available. They work for all editions of NLog. Additional appenders are available in separate assemblies which may or may not work for all platforms.

Appender Assembly Description
Console NLog.dll writes log messages to the console
File NLog.dll logs to a file
Null NLog.dll discards logging messages
Memory NLog.dll stores log messages in memory for programmatic retrieval
ASP.Net Trace NLog.ASPNet.dll ASP.NET Trace Appender

Common Appender Configuration

All appenders accept the following parameters:

Parameter Name Required Description
Name Yes The name of the appender. This should be a human-readable name. For example: console, masterlogfile.
Type Yes The type of the appender. This can be a simple type name (for appenders defined in NLog.dll) or a fully qualified type name.
Layout No The format of logged messages. See Layout Appenders for more info.

Console Appender

This is a very simple appender. It accepts no additional parameters beside common ones and its configuration is very simple. The following example outputs all messages to the console using a simple 4-column format:

<nlog> <appenders> <appender name="console" type="Console" layout="${longdate}|{$level}|${logger}|${message}" /> </appenders> <rules> <logger name="*" appendTo="console" /> </rules> </nlog>

File Appender

This appender is quite powerful. It lets you write your output to one or more files in an excusive or concurrent manner. The following is an example of the configuration which causes each log message to be written to a file named LOGGER_NAME.SHORT_DATE.log.

<nlog> <appenders> <appender name="logfile" type="File" layout="${longdate}|{$level}|${logger}|${message}"> <filename>c:\path_to\log\files\${logger}.${shortdate}.log</filename> </appenders> <rules> <logger name="*" appendTo="logfile" /> </rules> </nlog>
